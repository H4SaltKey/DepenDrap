# 🔐 アカウント・パスワードシステム

## 概要

ニックネーム（ユーザー名）とパスワードの組み合わせでアカウントを管理します。同じニックネームでも異なるパスワードなら別のアカウントとして作成できます。

---

## アカウント構造

### Firebase データ構造

```json
{
  "accounts": {
    "{nickname}": {
      "id": "abc123def456",
      "nickname": "ディペンド太郎",
      "password": "mypassword123",
      "createdAt": 1715000000000,
      "lastLogin": 1715000000000
    }
  },
  "players": {
    "{nickname}": {
      "status": {
        "isOnline": true,
        "lastSeen": 1715000000000,
        "sessionId": "xyz789..."
      }
    }
  }
}
```

### アカウント情報

| フィールド | 説明 | 例 |
|-----------|------|-----|
| `id` | アカウント ID（ユニーク） | `abc123def456` |
| `nickname` | ニックネーム（ユーザー名） | `ディペンド太郎` |
| `password` | パスワード | `mypassword123` |
| `createdAt` | 作成日時 | `1715000000000` |
| `lastLogin` | 最後のログイン日時 | `1715000000000` |

## ログイン・登録フロー

### ログイン

```
1. ログイン画面を開く
2. ニックネームを入力
3. パスワードを入力
4. 「LOGIN」をクリック
   ↓
5. Firebase で accounts/{nickname} を検索
6. パスワードを検証
7. 一致したらログイン成功
8. index.html へ遷移
```

**検証ルール：**
- ニックネーム：2文字以上
- パスワード：4文字以上
- アカウントが存在する必要がある
- パスワードが一致する必要がある

### 登録（新規アカウント作成）

```
1. ログイン画面で「REGISTER」をクリック
2. ニックネームを入力
3. パスワードを入力
4. 「CREATE ACCOUNT」をクリック
   ↓
5. Firebase で accounts/{nickname} を検索
6. 同じニックネーム + パスワードの組み合わせが存在するか確認
7. 存在しなければ新規作成
8. アカウント ID を生成
9. Firebase に保存
10. ログイン状態に設定
11. index.html へ遷移
```

**検証ルール：**
- ニックネーム：2文字以上
- パスワード：4文字以上
- 同じニックネーム + パスワードの組み合わせは作成不可
- 同じニックネームでも異なるパスワードなら作成可能

## 同じニックネームの複数アカウント

### 例

**アカウント 1：**
- ニックネーム：`ディペンド太郎`
- パスワード：`password123`

**アカウント 2：**
- ニックネーム：`ディペンド太郎`
- パスワード：`password456`

### 動作

```
ログイン画面：
  ニックネーム: ディペンド太郎
  パスワード: password123
  → アカウント 1 でログイン

ログイン画面：
  ニックネーム: ディペンド太郎
  パスワード: password456
  → アカウント 2 でログイン
```

### 注意

- Firebase では `accounts/{nickname}` をキーとしているため、同じニックネームの場合は最後に作成されたアカウントが上書きされます
- 複数アカウントを管理する場合は、異なるニックネームを使用することをお勧めします

## ログイン画面の使用方法

### ログインモード

```
タイトル: LOGIN
ボタン: LOGIN / REGISTER

入力項目:
  - Nickname: ディペンド太郎
  - Password: mypassword123

クリック: LOGIN
```

### 登録モード

```
タイトル: REGISTER
ボタン: CREATE ACCOUNT / LOGIN

入力項目:
  - Nickname: ディペンド太郎
  - Password: newpassword456

クリック: CREATE ACCOUNT
```

### モード切り替え

- 「REGISTER」ボタンをクリック → 登録モードに切り替え
- 「LOGIN」ボタンをクリック → ログインモードに切り替え

## Firebase Console での確認

### アカウント情報を確認

1. Firebase Console を開く
2. プロジェクトを選択
3. 「Realtime Database」を選択
4. 「データ」タブを確認
5. `accounts` → `{nickname}` を確認

**例：**
```json
{
  "accounts": {
    "ディペンド太郎": {
      "id": "abc123def456",
      "nickname": "ディペンド太郎",
      "password": "mypassword123",
      "createdAt": 1715000000000,
      "lastLogin": 1715000000000
    }
  }
}
```

### オンライン状態を確認

1. `players` → `{nickname}` → `status` を確認

**例：**
```json
{
  "players": {
    "ディペンド太郎": {
      "status": {
        "isOnline": true,
        "lastSeen": 1715000000000,
        "sessionId": "xyz789..."
      }
    }
  }
}
```

## セキュリティに関する注意

⚠️ **パスワード管理について**
- パスワードは平文で Firebase に保存されます
- 本番環境ではハッシュ化を推奨
- 強力なパスワードを使用してください

⚠️ **Firebase セキュリティルール**
- 現在はテストモードで、誰でも読み書き可能
- 本番環境ではセキュリティルールを設定してください

**推奨セキュリティルール：**
```json
{
  "rules": {
    "accounts": {
      "$nickname": {
        ".read": false,
        ".write": false
      }
    },
    "players": {
      "$nickname": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```
